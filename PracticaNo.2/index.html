Introducción

¿Qué es JavaScript y cuáles son sus principales usos en la web?

JavaScript, a menudo abreviado como JS, es un lenguaje de programación de alto nivel y un pilar muy importante para el desarrollo web. Diferente a HTML y CSS, que éstos se encargan de la estructura y el estilo de una página, JavaScript le otorga interactividad y dinamismo. Permite que las páginas web respondan a las acciones del usuario, manipulen contenido, envíen y reciban datos del servidor y creen animaciones complejas.

En resumen, los usos principales de JavaScript en la web son:

  • Interactividad: Validar formularios, crear menús desplegables y gestionar eventos como clics o movimientos del ratón
  • Manipulación del DOM: Modificar el contenido, estructura y estilo de una página web en tiempo real sin necesidad de recargarla.
  • Comunicación con el servidor(AJAX): Enviar solicitudes a servidores para obtener datos y actualizar partes de la página, como en las aplicaciones de "scroll infinito" o en los chats en tiempo real
  • Animaciones y efectos visuales: Crear carruseles de imágenes, transiciones y efectos visuales atractivos

¿En qué año y con qué propósito se creó JavaScript?

JavaScript fue creado en 1995 por Brendan Eich mientras trabajaba en Netscape Communications. Originalmente, se le llamó Mocha, luego LiveScript, y finalmente se cambió a JavaScript. El propósito inicial de su creacíon fue ser un lenguaje de scripting ligero para navegadores web, permitiendo a los desarrolladores y diseñadores agregar interactividad a las páginas HTML de una manera sencilla y rápida.

Su objetivo principal era liberar al servidor de la carga de procesar cada pequeña acción del usuario, como la validación de un formulario, y permitir que esas tareas se ejecutaran directamente en el navegador del cliente.

¿Por qué JavaScript no es lo mismo que Java?

Esta es una confusión muy común. Aunque sus nombres sean algo similares, JavaScript y Java son dos lenguajes de programación completamente diferentes. La inclusión de la palabra "Java" en el nombre de JavaScript fue una estrategia de marketing de Netscape en la década de 1990 para capitalizar la popularidad de Java en ese momento.

A continuación, una comparación para entender en qué son diferentes

¿Cómo se complementa JavaScript con HTML y CSS?

JavaScript, HTML y CSS forman el trío fundamental del desarrollo web. HTML se encarga de la estructura y el contenido de la página, definiendo elementos como títulos, párrafos, imágenes y enlaces. CSS se encarga de la presentación y el estilo, controlando colores, fuentes, diseño y animaciones.

JavaScript es el cerebro que da funcionalidad. Mientras HTML crea un botón y CSS lo hace bonito, JavaScript le dice qué hacer cuando alguien le hace clic: puede cambiar el texto, mostrar un mensaje, o incluso enviar una solicitud a un servidor. Trabajan juntos de manera que JavaScript interactúa con el HTML y CSS a través del DOM (Document Object Model) para modificar la página de forma dinámica.

Fundamentos del lenguaje

¿Cuál es la sintaxis básica de JavaScript y cómo se escriben comentarios?

La sintaxis de JavaScript es muy flexible y fácil de leer. Las instrucciones (sentencias) se separan con un punto y coma (;), aunque en la mayoría de los casos es opcional. Las sentencias se agrupan en bloques usando llaves {}.

Para los comentarios, que son líneas de texto que el intérprete de JavaScript ignora, existen dos formas:

  • Comentario de una sola línea: Se usa doble barra inclinada //. Ejemplo: // Esto es un comentario de una línea.
  • Comentario de múltiples líneas: Se inicia con /* y termina con */. Ejemplo: /* Esto es un comentario que se extiende a lo largo de varias líneas. */

¿Qué diferencias hay entre var, let y const?

Estas tres palabras clave se usan para declarar variables, pero tienen diferencias cruciales en cuanto a su alcance (scope) y si se pueden reasignar.

  • var: Es la forma tradicional de declarar variables. Su alcance es global o de función. Puede ser redeclarada y reasignada. Se considera obsoleta en el desarrollo moderno debido a que puede llevar a errores inesperados.
  • let: Se introdujo en ES6 (ECMAScript 2015). Su alcance es de bloque ({}). No se puede redeclarar en el mismo bloque, pero sí se puede reasignar. Es la forma preferida para declarar variables cuyo valor va a cambiar.
  • const: También se introdujo en ES6. Su alcance es de bloque. No se puede redeclarar ni reasignar. Se usa para declarar variables cuyo valor no va a cambiar, como constantes matemáticas o elementos del DOM. Sin embargo, si const se usa para un objeto o un array, el contenido del objeto o array sí se puede modificar, pero no se puede reasignar la variable a un nuevo objeto o array.

¿Cuáles son los tipos de datos primitivos en JavaScript?

Los tipos de datos primitivos son los tipos de datos más básicos y fundamentales en JavaScript. Los siete tipos de datos primitivos son:

  1. string (cadena de texto): Se usa para representar texto. Se encierra entre comillas simples, dobles o backticks.
    Ejemplo: let nombre = "Juan";
  2. number (número): Representa tanto números enteros como decimales.
    Ejemplo: let edad = 30;
  3. boolean (booleano): Solo puede tener dos valores: true (verdadero) o false (falso).
    Ejemplo: let esMayorDeEdad = true;
  4. undefined (indefinido): Indica que una variable ha sido declarada pero no se le ha asignado un valor.
    Ejemplo: let valor;
  5. null (nulo): Representa la ausencia intencional de un valor para una variable.
    Ejemplo: let carro = null;
  6. symbol (símbolo): Se usa para crear identificadores únicos.
    Ejemplo: const id = Symbol("id unico");
  7. bigint (número entero grande): Se usa para representar números enteros que son demasiado grandes para el tipo number.
    Ejemplo: let numeroGrande = 12345678901234567890n;

¿Qué diferencia hay entre un array y un objeto en JavaScript?

La principal diferencia radica en cómo organizan y acceden a sus datos.

  • Un array (arreglo) es una colección ordenada de elementos. Los elementos se acceden por su índice numérico, que comienza en 0. Piensa en él como una lista o una fila.
    Ejemplo: let colores = ["rojo", "verde", "azul"]; Aquí, "rojo" es el elemento en el índice 0.
  • Un objeto es una colección de pares clave-valor. En lugar de usar índices numéricos, los valores se acceden a través de una clave (un nombre, que es una cadena de texto). Piensa en él como un diccionario.
    Ejemplo: let persona = { nombre: "Ana", edad: 25 }; Aquí, "nombre" y "edad" son las claves.

¿Qué son y cómo se usan los operadores aritméticos, lógicos y relacionales?

Los operadores son símbolos especiales que realizan operaciones sobre los valores.

  • Operadores aritméticos: Realizan operaciones matemáticas.
    Ejemplos: + (suma), - (resta), * (multiplicación), / (división), % (módulo o residuo), ++ (incremento), -- (decremento).
  • Operadores lógicos: Se usan para combinar o invertir valores booleanos.
    Ejemplos: && (AND lógico), || (OR lógico), ! (NOT lógico).
    true && false da false.
    true || false da true.
    !true da false.
  • Operadores relacionales (o de comparación): Comparan dos valores y devuelven un valor booleano (true o false).
    Ejemplos: == (igualdad de valor), === (igualdad estricta de valor y tipo), != (diferente de valor), !== (diferente estricto), > (mayor que), < (menor que), >= (mayor o igual que), <= (menor o igual que).

Control de flujo

¿Cómo funcionan las sentencias condicionales if, else y switch?

Las sentencias condicionales permiten ejecutar bloques de código solo si se cumplen ciertas condiciones.

  • if / else: Ejecuta un bloque de código si una condición es true, y opcionalmente otro bloque si es false. 

    if (edad >= 18) {
  console.log("Eres mayor de edad.");
} else {
  console.log("Eres menor de edad.");
}

    También se puede usar else if para múltiples condiciones.
  • switch: Se usa para evaluar una expresión y ejecutar un bloque de código diferente dependiendo del valor de esa expresión. Es más legible que una larga cadena de if...else if para múltiples opciones. 

    switch (dia) {
  case "lunes":
    console.log("Inicio de semana.");
    break;
  case "viernes":
    console.log("¡Por fin es viernes!");
    break;
  default:
    console.log("Es otro día.");
}

¿Cuáles son las diferencias entre los bucles for, while y do...while?

Los bucles se usan para repetir un bloque de código.

  • for: Es el más común para iterar un número conocido de veces. Incluye la inicialización, la condición y el incremento en la misma línea. 

    for (let i = 0; i < 5; i++) {
  console.log(i); // Imprime 0, 1, 2, 3, 4
}
  • while: Repite un bloque de código mientras una condición sea verdadera. La condición se evalúa antes de cada iteración. 

    let i = 0;
while (i < 5) {
  console.log(i);
  i++;
}
  • do...while: Es similar a while, pero garantiza que el bloque de código se ejecute al menos una vez, ya que la condición se evalúa al final de la iteración. 

    let i = 5;
do {
  console.log(i); // Imprime 5
  i++;
} while (i < 5);

¿Para qué sirven las palabras clave break y continue?

Estas palabras clave permiten controlar el flujo dentro de un bucle.

  • break: Detiene y sale completamente del bucle actual, sin importar si la condición del bucle sigue siendo verdadera. Se usa para terminar un bucle prematuramente.
  • continue: Salta la iteración actual y pasa a la siguiente. El bucle no se detiene, simplemente ignora el resto del código dentro del bloque para esa iteración.

Funciones

¿Cómo se declara una función y cómo se llama en JavaScript?

Una función es un bloque de código reutilizable que realiza una tarea específica.

  • Declaración de función: 

    function saludar() {
  console.log("Hola a todos.");
}
  • Llamada a la función: Para ejecutar el código dentro de la función, simplemente se usa su nombre seguido de paréntesis. 

    saludar(); // Imprime "Hola a todos."

¿Qué significa que una función tenga parámetros y un valor de retorno?

  • Parámetros: Son variables que se especifican en la declaración de la función y que actúan como "contenedores" para los valores que se le pasan cuando se la llama. Los valores que se le pasan se llaman argumentos. 

    function sumar(a, b) { // 'a' y 'b' son parámetros
  return a + b;
}
let resultado = sumar(5, 3); // 5 y 3 son argumentos
  • Valor de retorno: Una función puede devolver un valor después de que ha terminado de ejecutarse. Para ello, se usa la palabra clave return. Si una función no tiene un return, devuelve undefined por defecto.

¿Qué son las funciones flecha (=>) y qué ventajas tienen?

Las funciones flecha, introducidas en ES6, son una sintaxis más concisa y corta para definir funciones. Se ven así: () => {}.

  • Sintaxis tradicional: 

    function miFuncion() {
  // ...
}
  • Sintaxis de función flecha: 

    const miFuncion = () => {
  // ...
}

Ventajas:

  • Sintaxis concisa: Son más cortas y fáciles de leer para funciones simples.
  • this léxico: A diferencia de las funciones tradicionales, las funciones flecha no crean su propio this. Esto significa que heredan el valor de this de su contexto circundante, lo que soluciona muchos problemas comunes con el this en JavaScript.

¿Qué es el scope en JavaScript y cómo afecta a las variables?

El scope o ámbito es la visibilidad de las variables. Define dónde las variables y funciones son accesibles.

  • Scope global: Las variables declaradas fuera de cualquier función o bloque tienen un alcance global, lo que significa que se puede acceder a ellas desde cualquier parte del código.
  • Scope de función: Las variables declaradas con var dentro de una función solo son accesibles dentro de esa función.
  • Scope de bloque: Las variables declaradas con let o const dentro de un bloque de código ({}) solo son accesibles dentro de ese bloque.

Comprender el scope es fundamental para evitar conflictos y errores en el código, ya que una variable en un scope no puede ser accedida desde otro scope si no está en el mismo nivel o superior.

Objetos y colecciones

¿Cómo se declara un objeto y cómo se accede a sus propiedades?

Los objetos se declaran usando llaves {}. Dentro de las llaves, se especifican pares de clave: valor.

  • Declaración: 

    let persona = {
  nombre: "María",
  edad: 28,
  ciudad: "Madrid"
};
  • Acceso a las propiedades: Se puede acceder a las propiedades de un objeto de dos formas:
    1. Notación de punto: Es la más común y legible.
      Ejemplo: console.log(persona.nombre); // Imprime "María"
    2. Notación de corchetes: Se usa cuando la clave es una variable o contiene caracteres especiales.
      Ejemplo: console.log(persona['edad']); // Imprime 28

¿Qué diferencia hay entre una propiedad y un método en un objeto?

  • Una propiedad es una variable o un valor asociado a un objeto. Simplemente almacena un dato.
    Ejemplo: En el objeto persona, nombre es una propiedad.
  • Un método es una función que está asociada a un objeto. Un método realiza una acción sobre los datos del objeto o los utiliza de alguna manera.
    Ejemplo: Podríamos agregar un método al objeto persona para que se presente a sí misma. 

    let persona = {
  nombre: "María",
  saludar: function() { // saludar es un método
    console.log("Hola, soy " + this.nombre);
  }
};
persona.saludar(); // Llamamos al método

Menciona al menos 3 métodos básicos de los arrays y explica cómo funcionan

Los arrays tienen una gran cantidad de métodos integrados para manipularlos. Aquí hay tres ejemplos muy comunes:

  1. push(): Agrega uno o más elementos al final de un array y devuelve la nueva longitud del array.
    Ejemplo: 

    let frutas = ["manzana", "banana"];
frutas.push("naranja"); // frutas ahora es ["manzana", "banana", "naranja"]
  2. pop(): Elimina el último elemento de un array y lo devuelve.
    Ejemplo: 

    let frutas = ["manzana", "banana", "naranja"];
let frutaEliminada = frutas.pop(); // frutaEliminada es "naranja", frutas es ["manzana", "banana"]
  3. map(): Crea un nuevo array con los resultados de la llamada a una función para cada elemento del array. Es muy útil para transformar arrays.
    Ejemplo: 

    let numeros = [1, 2, 3];
let dobles = numeros.map(function(num) {
  return num * 2;
}); // dobles ahora es [2, 4, 6]

Otros temas básicos

¿Qué es el DOM y por qué es importante en el uso de JavaScript en páginas web?

El DOM (Document Object Model) es una interfaz de programación para documentos HTML y XML. Es una representación de la estructura del documento como un árbol de nodos, donde cada nodo representa una parte del documento (como un elemento, un atributo o un texto).

Su importancia para JavaScript es crucial: el DOM permite que JavaScript acceda y manipule el contenido y la estructura de una página web. Sin el DOM, JavaScript no podría cambiar dinámicamente el texto de un párrafo, agregar un nuevo elemento a una lista, o cambiar el color de un botón. En pocas palabras, el DOM es el puente que permite a JavaScript interactuar con HTML y CSS.

¿Qué es un evento en JavaScript y menciona al menos 3 ejemplos comunes?

Un evento es una acción o suceso que ocurre en un documento HTML y que el navegador puede detectar. Los eventos son la forma en que JavaScript maneja las interacciones del usuario. Cuando ocurre un evento (como un clic o una pulsación de tecla), JavaScript puede ejecutar una función específica en respuesta.

Ejemplos comunes de eventos:

  • click: Ocurre cuando el usuario hace clic en un elemento.
  • mouseover: Ocurre cuando el puntero del ratón se mueve sobre un elemento.
  • submit: Ocurre cuando se envía un formulario.
  • keydown: Ocurre cuando el usuario presiona una tecla.

¿Cuál es la diferencia entre alert, prompt y console.log?

  • alert(): Muestra una pequeña ventana de diálogo con un mensaje al usuario. Se usa principalmente para avisos o notificaciones simples. El usuario debe hacer clic en "Aceptar" para continuar. Es intrusivo y detiene la ejecución del código hasta que se cierra.
  • prompt(): Muestra una ventana de diálogo que le pide al usuario que ingrese un valor. La función devuelve el valor ingresado como una cadena de texto. También detiene la ejecución del código.
  • console.log(): Es una de las herramientas más utilizadas por los desarrolladores. Muestra un mensaje en la consola del navegador. Se usa principalmente para depurar código y ver el valor de las variables sin afectar la interfaz de usuario. A diferencia de alert y prompt, no interrumpe el flujo de ejecución del código.

¿Cómo se abre la consola del navegador y qué ventajas tiene usarla para depurar código?

Para abrir la consola del navegador, generalmente se usan los siguientes atajos de teclado:

  • En Chrome, Firefox, Edge, etc.: F12 o Ctrl + Shift + I (Windows) / Cmd + Opt + I (Mac).
  • Una vez que se abre la "Herramientas para desarrolladores", la pestaña Console es la que nos interesa.

Ventajas de usar la consola para depurar:

  • Ver errores: La consola muestra errores de sintaxis y de tiempo de ejecución, lo que ayuda a identificar rápidamente dónde está el problema en el código.
  • Inspeccionar valores: Puedes usar console.log() para imprimir el valor de las variables, objetos o el resultado de una función en cualquier punto de tu código. Esto es invaluable para verificar si los datos están fluyendo como esperas.
  • Pruebas rápidas: Puedes escribir y ejecutar código JavaScript directamente en la consola para probar una función o una lógica sin necesidad de recargar la página.
